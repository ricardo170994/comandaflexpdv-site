Por que essa pagina existe

O projeto merece ser apresentado como produto.

O Comanda Flex PDV nao e so um CRUD de pedidos. Ele entrega operacao, governanca, continuidade e visibilidade comercial em uma estrutura unica.

Fluxo

Pedido sai do salao e chega na producao com menos ruído

Categoria, busca, extras, notas rapidas, delivery, promocao automatica e validacao de impressora antes da gravacao.

Confiabilidade

O modulo do garcom monitora conectividade, guarda pedidos em fila local e sincroniza quando o servidor volta.

Cozinha

SLA visivel, historico auditavel e alerta sonoro de novo pedido

Financeiro

Caixa com abertura, reforco, sangria, divergencia e relatorio

O recebimento aceita pagamentos parciais, respeita troco por metodo e emite comprovante com reimpressao rastreada.

Gestao

Continuidade

Base tecnica

Produto funcional com stack objetiva e implantacao simples.

A pagina tambem comunica maturidade tecnica: o projeto ja nasceu com bootstrap, instalador, PDF, QR local e separacao clara de modulos.

Flask + Jinja + JavaScript

Estrutura web direta, leve e facil de manter para uma operacao que precisa rodar sem um front-end pesado.

SQLite com bootstrap automatico

Ao iniciar, o sistema cria base, schema, seed e recupera admin generico quando necessario.

Waitress, QR local e acesso rapido

O servidor pode subir com QR de acesso pela rede local para acelerar uso em terminais moveis.

PDF, impressao e identidade visual

ReportLab, Pillow e configuracao de recibo permitem padronizar documentos, vias e ativos do estabelecimento.

i18n, fuso e configuracao por operacao

O sistema contempla linguagem, timezone, performance preset e preferencias voltadas ao contexto real do restaurante.

Empacotamento Windows pronto

A pipeline ja considera PyInstaller, Inno Setup e metadados centralizados para gerar EXE e instalador.

Publicacao

A pagina ja esta pronta para ir ao ar.

O fluxo mais simples e subir o repositorio e usar o GitHub Actions para publicar o conteudo de docs/ como site oficial.

O que foi preparado

  • Landing page com narrativa comercial e modulos interativos.
  • CSS proprio, responsivo e com identidade alinhada ao projeto.
  • JavaScript para tabs, animacao de entrada e contador.

Como colocar no ar

  1. Envie a branch principal para o GitHub.
  2. Em Settings > Pages, selecione GitHub Actions.
  3. O workflow publica automaticamente a pasta docs/.
